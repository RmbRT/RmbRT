I am a performance-wing extremist and a great machine-oriented programming enthusiast, and I believe in *the right thing* (as opposed to the subversive *worse is better*).
I currently work at [PolyCrypt GmbH](https://polycry.pt) as a systems architect and project planner since 2026. Previously, I was a core developer, working on our state channel framework [go-perun](https://github.com/perun-network/go-perun) and our L2 platform [Erdstall](https://erdstall.dev) for 7 years (2019–2025).
In my free time, I work on [my own programming language](https://github.com/RmbRT-lang) (self-hosted compiler is a WIP, perf. goal: 1M LOC/s), a CPU ISA, a graphics API, and an OS.

My dream is to reenvision computing from the ground up, making it as programmer-friendly, hardware-friendly, simple, and efficient as possible:

* A 2GiB RAM machine with an iGPU should be entirely sufficient for all personal computing, and all programs should be maintainable by a single person. 4GiB RAM should be considered abundant.
* An operating system should be maintainable by a single person.
* An ISA should properly specify access to all common hardware periphery (what the x86 BIOS did, but for network, GPU, sound, disk, microphone, camera, mouse, keyboard), and drivers should not need to exist except for exotic/non-standard functionality or uncommon device types. A kernel for such an ISA should thus also be a reasonably small project maintainable by a single person.
* Programs should ship as source archives and should be compiled on the user machine at launch, at 1M LOC/s.
* Software should be eternal.
